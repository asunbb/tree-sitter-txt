# tree-sitter-txt 代码审核与改进计划

## Context

项目目标：用 2 空格缩进拆分段落，为编辑器提供折叠功能。项目已经过 3 轮迭代（初始版本→折叠判断修正→缩进栈追踪），核心逻辑已基本稳定。本次审核关注代码是否足够简单、直接、清晰，以及是否有改进空间。

---

## 审核结论

### grammar.js — 优秀，无需改动

语法定义已经是最简形式：
- 3 条规则（document / segment / content）
- 3 个外部 token（newline / indent / dedent）
- 递归结构清晰，一眼能看懂层级关系

**结论**：逻辑已经足够简单直接，没有改进空间。

---

### scanner.c — 有 3 处可改进

#### 问题 1：缩进级别只入栈一次，多级缩进跳跃产生错误 AST

**当前行为**（第 145-153 行）：
```
0 空格 → 6 空格（跳跃 3 级）
scanner 只 push 一个 new_level=3 到栈上
AST 结果：0→3 直接嵌套，跳过了 1、2 级
```

**分析**：这其实不算 bug——grammar 只要求 `_indent` 标记缩进增加，scanner 在栈上记录的是**实际**缩进级别。如果用户从 0 空格跳到 6 空格，AST 确实应该只有一层嵌套。这与 Python 的行为一致（Python 也不允许跳跃缩进）。

**结论**：当前行为合理，保持现状。但建议在 scanner.c 顶部加一行注释说明这个设计决策。

#### 问题 2：`newline_after_dedent` 标志的必要性值得审视

**当前逻辑**（第 87-95 行 + 第 164 行）：
- dedent 时设置 `newline_after_dedent = true`
- 下次 scan 优先发出一个零宽 NEWLINE token

**分析**：这个标志的目的是让 dedent 后的同行段落能正确接收 NEWLINE 分隔。这是对 grammar 中 `seq($._newline, $.segment)` 的配合。去掉它会导致 dedent 后的同级段落无法分隔。逻辑是必要的。

**结论**：保留。但可以加注释说明为什么需要这个标志。

#### 问题 3：序列化/反序列化的手动 memcpy 可读性差

**当前代码**（第 36-74 行）：手动管理 offset 和 memcpy，容易出错。

**分析**：但这是 tree-sitter scanner 的标准写法，几乎所有外部 scanner 都这样写。改为其他方式反而增加复杂度。

**结论**：保持现状，这是 tree-sitter 生态的惯例。

---

### folds.scm — 优秀，无需改动

从 `(segment) @fold` 改为 `[(segment)] @fold` 只是格式差异，效果一样。

## 改进计划

### 改动 1：为 scanner.c 关键决策点添加注释

**文件**：`src/scanner.c`

在以下位置添加简要注释：
1. 第 142-143 行附近：说明 "2 空格 = 1 级" 的设计决策，以及跳跃缩进的处理方式
2. 第 87 行附近：说明 `newline_after_dedent` 标志的存在原因
3. 第 118-139 行附近：说明空行跳过逻辑

### 改动 2：补充边界测试用例

**文件**：`test/corpus/basic.txt`

增加以下测试场景：
1. **空行分隔**：段落间有空行时的解析
2. **末尾空行**：文件末尾有多个空行
3. **首行缩进**：文件首行有缩进（非零级）

### 改动 3：统一命名

**现状**：grammar name 是 `litetxt`，package name 是 `tree-sitter-txt`。

**建议**：这是个人项目，影响不大。但如果在意一致性，可以统一。不改也没关系。

---

## 不建议改动的地方

1. **序列化/反序列化**：tree-sitter 标准写法，改动无收益
2. **MAX_INDENT_STACK 128**：远超实际需要，但改动无收益
3. **grammar 递归结构**：已经是最简形式
4. **content 正则 `/[^\n]+/`**：精确匹配"一行文本"，无法再简化

---

## 验证方式

1. `tree-sitter generate` 确认生成成功
2. `tree-sitter test` 确认所有测试通过（包括新增用例）
3. 手动测试折叠效果：`echo -e "A\n  B\n  C\nD" | tree-sitter parse /dev/stdin`
