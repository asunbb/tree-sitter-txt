# tree-sitter-txt-again

[中文](README.zh-CN.md)

A [tree-sitter](https://tree-sitter.github.io/) grammar (named **litetxt**) that parses indented text files and enables code folding for `.txt` files in editors.

## Features

- **Indentation-based segment parsing** — content at different indent levels is structured as parent-child segments
- **Multi-level nesting** — arbitrary nesting depth supported
- **Irregular indentation handling** — correctly processes code-block-like indentation patterns
- **Editor folding support** — includes fold queries (`queries/folds.scm`) for Neovim, VS Code, and other tree-sitter-compatible editors

## Quick Start

### Prerequisites

- [Node.js](https://nodejs.org/)
- [tree-sitter CLI](https://tree-sitter.github.io/tree-sitter/creating-parsers#installation) (`tree-sitter-cli`)

### Generate parser

```bash
npm run generate
```

### Run tests

```bash
npm test
```

### Parse text from stdin

```bash
echo "Heading
  Sub item 1
  Sub item 2
    Nested detail" | npm run parse
```

## Grammar

The grammar defines three core rules:

| Rule | Description |
|------|-------------|
| `document` | Root node containing zero or more segments |
| `segment` | A content line, optionally followed by indented child segments |
| `content` | Raw text matching `/[^\n]+/` |

Indentation handling is implemented via external tokens in `src/scanner.c`:

| Token | Purpose |
|-------|---------|
| `_newline` | Line break between sibling segments |
| `_indent` | Increase in indentation level |
| `_dedent` | Decrease in indentation level |

## Neovim Installation

### lazy.nvim

```lua
{
  "asunbb/tree-sitter-txt",
  dependencies = {"nvim-treesitter/nvim-treesitter"},
  config = function()
    require("litetxt").setup()
  end,
}
```

After installation, open a `.txt` file and run `:TSInstall litetxt` to compile the parser.
