# tree-sitter-txt

[简体中文](README.zh-CN.md)

A [tree-sitter](https://tree-sitter.github.io/) grammar whose parser is named **litetxt**, providing code folding for `.txt` files.

This repository provides parser updates as a plugin.

Note: the [nvim-treesitter](https://github.com/nvim-treesitter/nvim-treesitter) repository has been archived and is no longer updated (unless the original author reverses the action).

## Features

- **Indentation-based segment parsing** — content at different indent levels is structured as parent-child segments
- **Multi-level nesting** — arbitrary nesting depth supported
- **Irregular indentation handling** — correctly processes code-block-like indentation patterns
- **Editor folding support** — includes fold queries (`queries/folds.scm`)

## Grammar

The grammar defines three core rules:

| Rule | Description |
|------|-------------|
| `document` | Root node containing zero or more segments |
| `segment` | A content line, optionally followed by indented child segments |
| `content` | Raw text matching `/[^\n]+/` |

Indentation handling is implemented via external tokens in `src/scanner.c`:

| Token | Purpose |
|-------|---------|
| `_newline` | Line break between sibling segments |
| `_indent` | Increase in indentation level |
| `_dedent` | Decrease in indentation level |
