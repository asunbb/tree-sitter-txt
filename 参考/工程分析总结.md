# tree-sitter-txt 工程分析总结

## 一、工程概述

| 属性 | 值 |
|------|-----|
| 名称 | tree-sitter-txt |
| 版本 | 0.1.0 |
| 许可证 | MIT |
| 描述 | Tree-sitter grammar for indented text segments |
| 用途 | 为 `.txt` 文件生成 tree-sitter parser，用于文本编辑器中的**代码折叠（fold）** 功能 |
| 作用域 | `source.txt` |
| 开发依赖 | tree-sitter-cli ^0.24.7 |

该工程通过解析缩进文本的层级关系，将纯文本转换为结构化的 AST（抽象语法树），使得文本编辑器（如 Neovim、VS Code 等）能够基于 tree-sitter 提供**折叠**支持。

## 三、核心文件分析

### 3.1 grammar.js — 语法定义

路径：`grammar.js`

```javascript
module.exports = grammar({
  name: "txt",

  externals: ($) => [$._newline, $._indent, $._dedent],

  rules: {
    document: ($) => repeat($.segment),
    segment: ($) =>
      seq(
        $.content,
        optional(
          seq($._indent, $.segment, repeat(seq($._newline, $.segment)), $._dedent)
        )
      ),
    content: ($) => /[^\n]+/,
  },
});
```

**语法规则解析**：

- **document**（文档根节点）：由零或多个 `segment` 组成
- **segment**（段落节点）：
  - 必须包含一个 `content`（文本内容）
  - 可选包含缩进子段：`_indent` + 第一个子 `segment` + 零或多个（`_newline` + 同级 `segment`）+ `_dedent`
- **content**（内容节点）：匹配除换行符外的任意字符，即一行文本
- **外部 token**（由 scanner.c 处理）：
  - `_newline` — 同级段落之间的换行分隔
  - `_indent` — 缩进增加标记
  - `_dedent` — 缩进减少标记

### 3.2 src/scanner.c — 外部扫描器

路径：`src/scanner.c`

这是手写的 C 语言扫描器，负责处理缩进检测——这是 tree-sitter 内置语法无法处理的上下文敏感逻辑。

**核心机制**：

| 功能 | 实现方式 |
|------|----------|
| 缩进单位 | **2 个空格 = 1 级缩进** |
| 状态跟踪 | `indent_level`（当前缩进级别）+ `pending_dedents`（待发出的 DEDENT 数量）|
| 空行处理 | 跳过空行，继续扫描下一行非空内容的缩进 |
| EOF 处理 | 文件末尾时逐级发出剩余的 DEDENT token |
| 序列化/反序列化 | 支持增量解析时的状态保存与恢复 |

**扫描逻辑流程**：

1. 若有待发出的 DEDENT，优先发出
2. 到达 EOF 时，发出剩余 DEDENT 逐级归零
3. 遇到换行符时：
   - 跳过连续空行
   - 统计下一行非空内容的前导空格数
   - 计算 `new_level = spaces / 2`
   - 若 `new_level > indent_level` → 发出 INDENT
   - 若 `new_level < indent_level` → 发出 DEDENT（可能需要多次）
   - 若 `new_level == indent_level` → 发出 NEWLINE

### 3.3 queries/folds.scm — 折叠查询

路径：`queries/folds.scm`

```scheme
(segment) @fold
```

这一行查询表示：编辑器可以对所有 `segment` 节点执行折叠操作。由于 `segment` 是递归嵌套的，因此支持**多级折叠**。

## 四、AST 节点类型

路径：`src/node-types.json`

| 节点类型 | 命名 | 角色 | 子节点 |
|----------|------|------|--------|
| `document` | 是 | 根节点 | 0..n 个 `segment` |
| `segment` | 是 | 段落节点 | 1 个 `content` + 0..n 个 `segment` |
| `content` | 是 | 内容节点 | 无子节点（叶节点） |

**层级关系**：

```
document
├── segment
│   ├── content (文本内容)
│   ├── segment (缩进子段)
│   │   ├── content
│   │   └── segment (更深层子段)
│   │       └── ...
│   └── segment (同级子段)
│       └── content
└── segment
    └── content
```

## 五、构建与测试

### npm scripts

| 命令 | 作用 |
|------|------|
| `npm run generate` 或 `npm run gen` | 根据 `grammar.js` 生成 `parser.c`、`grammar.json`、`node-types.json` |

```bash
# 1. 安装依赖
npm install

# 2. 生成 parser
tree-sitter generate

# 3. 运行测试
tree-sitter test

# 4. 测试解析效果
echo -e "Parent\n  Child" | tree-sitter parse /dev/stdin
```

## 六、工作原理

### 解析流程

```
输入文本                    Scanner (scanner.c)          Parser (parser.c)
─────────                   ──────────────────           ─────────────────
"Parent\n"                  
  → 遇到换行                 
  → 统计下一行前导空格        
"  Child\n"                 
  → 2 空格 > 0 级           
  → 发出 INDENT  ──────→    INDENT token
                             ─────────────→  解析 grammar.js 中的规则
"Back\n"                                         构建 AST 树
  → 0 空格 < 1 级           
  → 发出 DEDENT  ──────→    DEDENT token
                             ─────────────→  关闭子 segment 节点
EOF                           DEDENT token
                             ─────────────→  关闭所有未关闭的节点
```

### 折叠工作流程

```
文本编辑器打开 .txt 文件
    ↓
tree-sitter 使用本 parser 生成 AST
    ↓
编辑器加载 queries/folds.scm
    ↓
匹配所有 (segment) 节点标记为 @fold
    ↓
用户可以对任意 segment 节点执行折叠/展开操作
```

## 七、测试用例说明

路径：`test/corpus/basic.txt`，共 4 个测试用例：

### 7.1 Simple flat segments（平级段落）

**输入**：
```
First segment
Second segment
Third segment
```

**AST**：
```
(document
  (segment (content))
  (segment (content))
  (segment (content)))
```

验证同级段落无嵌套，每个 segment 仅有 content，无子 segment。

### 7.2 Nested segments - one level（一级嵌套）

**输入**：
```
Parent
  Child A
  Child B
```

**AST**：
```
(document
  (segment
    (content)
    (segment (content))
    (segment (content))))
```

验证一级缩进产生父子关系，Parent 下有两个子 segment。

### 7.3 Multi-level nesting（多级嵌套）

**输入**：
```
Level 0
  Level 1
    Level 2
Back to level 0
```

**AST**：
```
(document
  (segment
    (content)
    (segment
      (content)
      (segment (content))))
  (segment (content)))
```

验证三级嵌套（0→1→2），以及返回 0 级时正确关闭中间层级。

### 7.4 Mixed nesting（混合嵌套）

**输入**：
```
A
  A1
  A2
    A2a
B
  B1
C
```

**AST**：
```
(document
  (segment
    (content)
    (segment (content))
    (segment
      (content)
      (segment (content))))
  (segment
    (content)
    (segment (content)))
  (segment (content)))
```

验证复杂混合场景：不同深度的分支、多顶级段落、不同子段数量，覆盖了实际使用中的典型场景。

## 八、总结

tree-sitter-txt 是一个精简而实用的 tree-sitter 语法包，核心只有两个文件：

- **grammar.js**（约 20 行）— 定义三层递归结构
- **scanner.c**（约 130 行）— 处理缩进检测

配合一行折叠查询 `(segment) @fold`，即可让文本编辑器对缩进文本提供完整的折叠能力。这种设计充分利用了 tree-sitter 的外部扫描器机制，将上下文敏感的缩进逻辑从声明式语法中分离出来，保持了语法定义的简洁性。
